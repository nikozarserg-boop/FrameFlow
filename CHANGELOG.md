# Журнал изменений

## [0.1.5] - 2026-03-01

### Добавления
- Добавлена полная локализация на русский язык всех markdown-документов (README.md, GUIDE.md, CHANGELOG.md, QA_CHECKLIST.md).
- Добавлена поддержка темной темы для интерфейса приложения.
- Добавлена поддержка мультиязычности (i18n) в приложении.
- **Улучшения алгоритма сглаживания мыши** (5 ключевых оптимизаций):
  - **Jitter Detection**: Обнаружение и удаление микродрожания мыши (±1.5px), сохранение кликов.
  - **Bilateral Filter**: Двусторонняя фильтрация с сохранением резких переходов движения, удаление высокочастотного шума.
  - **Velocity-Aware Smoothing**: Адаптивное сглаживание по скорости — низкие скорости сильнее сглаживаются, высокие скорости сохраняют детали.
  - **Catmull-Rom with Acceleration**: Интерполяция с учетом ускорения для более плавных кривых между точками.
  - **Pre-Click Pattern Analysis**: Анализ движения перед кликом (dwell time, deceleration, jitter detection) для оптимизации поведения камеры.
- **Многоступенчатый конвейер обработки курсора**: Новая архитектура с 6 последовательными этапами фильтрации (вместо 3-4 в старой версии).
- **MotionPattern структура**: Типизированное представление паттерна движения перед кликом для интеграции с Smart Camera Engine.
- **Функция `analyze_pre_click_pattern()`**: Позволяет камере адаптировать поведение на основе типа клика пользователя.

### Изменения
- Переименование проекта с `NeuroScreenCaster` на `FrameFlow`.
  - Обновлен идентификатор приложения: `com.neuroscreencaster.app` → `com.frameflow.app`.
  - Обновлены пакеты Cargo: `neuroscreencaster` → `frameflow`, `neuroscreencaster_lib` → `frameflow_lib`.
  - Переименованы пути проектов: `~/Videos/NeuroScreenCaster/` → `~/Videos/FrameFlow/`.
  - Обновлены пути в конфиге Tauri: `$VIDEO/NeuroScreenCaster/**` → `$VIDEO/FrameFlow/**`.
  - Обновлены все ссылки в документации и конфигурационных файлах.
  - Обновлены сценарии сборки (build.bat, scripts/sync-version.mjs, scripts/qa-smoke.mjs).
- Оптимизация структуры проекта:
  - Удалены неиспользуемые иконки (Square*.png, StoreLogo.png, icon.png, 64x64.png).
  - Удалены иконки и конфигурация для неподдерживаемых платформ (iOS, Android, macOS).
  - Удалена favicon.png (не применима для Windows desktop приложения, используется только в веб-браузерах).
  - Оптимизирована папка `src-tauri/icons/` — остались только 4 нужных файла (32x32.png, 128x128.png, 128x128@2x.png, icon.ico).
  - Удален неиспользуемый Python скрипт генерации иконок (используется встроенная команда Tauri: `npx @tauri-apps/cli icon`).
  - Resample частота снижена с 120Hz на 60Hz для лучшей фильтрации без потери качества.
  - Параметры фильтрации переведены на динамические расчеты вместо статических значений.
  - Улучшена стабильность траектории курсора при переходах между медленными и быстрыми движениями.
  - **Оптимизация движений камеры**: снижена жесткость spring с 170 до 100, увеличена инертия (масса 1.5), усилено затухание — движения камеры стали более плавными и мягкими.
  - **Динамическое масштабирование редактора**: размеры элементов и отступы подстраиваются под размер экрана пользователя (clamp функции, vw единицы).
  - **Масштабирование превью**: добавлена поддержка Ctrl+колесо мыши для масштабирования превью (0.5x - 3.0x).
  - **Адаптивный UI редактора**: боковая панель и элементы переиспользуют от 20-25% ширины экрана, toolbar обертывается на узких экранах.
  - **Responsive дизайн**: все gap, padding, font-size используют clamp() для плавного масштабирования вместо фиксированных значений.

## [0.1.4] - 2026-02-23

### Исправления
- Добавлена надежная резервная копия прогресса экспорта, так что прогресс больше не остается на 0% до завершения в некоторых окружениях FFmpeg.
- Добавлена поддержка отмены экспорта (`cancel_export`) с корректным завершением ffmpeg и финальным статусом `Экспорт отменен`.
- Добавлены индикаторы ETA/Elapsed в UI экспорта и точное отображение прогресса (`x.y%`).
- Исправлена деградация экспорта на длинных видео: зумы и курсор больше не становятся дерганными из-за переупрощения траекторий.
- Для экспорта внедрено адаптивное motion-aware прореживание точек камеры и курсора (детализация растет с длительностью проекта, повороты и ускорения сохраняются лучше).
- Исправлена ошибка предпросмотра в редакторе `Failed to play video: AbortError`.
- Убрана гонка между `video.play()` и `video.pause()` при быстрых переключениях воспроизведения.
- Добавлена защита от устаревших `play()`-запросов при смене источника/состояния плеера.
- Ожидаемые браузерные `AbortError` больше не показываются как пользовательская ошибка.

## [0.1.3] - 2026-02-23

### Исправления
- Исправлен режим триггера автозума `1 клик`: клики больше не теряются из-за дефолтной кластеризации и большого `min_zoom_interval`.
- Для режимов `1 клик` и `Ctrl+клик` ослаблены внутренние debounce/cluster-ограничения, чтобы камера реагировала почти на каждый валидный клик.
- Исправлен кейс no-op переходов в строгих click-режимах: если семантическая цель дает `1x`, применяется fallback-зум от точки клика.
- В strict click-режимах отключено подавление ретаргета через safe-zone, из-за которого часть кликов раньше не меняла фокус.

## [0.1.2] - 2026-02-23

### Исправления
- Исправлен скролл в `LockedFocus`: сдвиг камеры теперь считается в корректной размерности (с учетом размеров экрана и текущего зума), без резких скачков.
- Исправлен `phantom escape` в `LockedFocus`: выход из фокуса теперь проверяется по выходу курсора за текущий viewport камеры, а не по простой дистанции до центра.
- Исправлено игнорирование быстрых кликов в автозуме: переходы больше не отбрасываются по `min_zoom_interval_ms`, а откладываются по времени старта.
- Исправлено дрожание на hard-edge панорамировании: шаг камеры ограничен фактическим пробоем границы.
- Сегменты зума на таймлайне синхронизированы с реальным движением spring-камеры (визуальные начало/конец лучше совпадают с фактическим зумом).
- Исправлено отображение времени на таймлайне: расчет ведется по реальной ширине lane (без колонки labels), устранен постоянный сдвиг playhead/сегментов.
- Усилены ограничения перетаскивания сегментов: добавлены neighbor-aware границы и минимальный зазор, чтобы сегменты не пересекались.
- Стабилизирована отрисовка сегментов при большом zoom таймлайна: соседние сегменты больше не тянут друг друга при редактировании границ.
- Добавлен единый тайминг-сдвиг курсора в предпросмотре и экспорте, чтобы уменьшить визуальное отставание курсора.
